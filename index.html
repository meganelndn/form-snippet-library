Input types

The HTML input element is used to create interactive controls for web-based forms and specifies an input field where the user can enter data. An input field can vary in many ways, depending on the type attribute.


Label should always be used with input fields to determine what input should be in them.
For points to the inputs ID.
Input type decides what content can be put inside the input.
Placeholder holds text that wil be hiddenn when the user starts to type in the field.
Required is an attribute you use, if the field is mandatory.
Pattern tells the input field what type of characters are valid.
Autofocus sets this input field in focus.

Forms

Inside of a form all types of inputs/selects can be placed.
Action attribute = what happens when "submit" is clicked.

Text

Elements of this input create basic single-line text fields and is used for text.

Number

Elements of this input type creates text fields and accepts numbers; it has an up/down switch.

Email

Elements of this input type creates text fields and accepts an email address.

URL

Elements of this input type create text fields that accepts a valid website URL.

Password

Elements of this input type create text fields that masks the text that is being entered with bullets.

Tel

Elements of this input type create text fields that accepts telephone numbers. There is no restricted format to accommodate all different countries.

Time

Elements of this input type create text fields that accepts time: hours, minutes, seconds.

Date

Elements of this input type create text fields that accepts dates; a calendar picker.

Range

Elements of this input type create text fields for selecting from a range of numbers.

Color

Elements of this input type create color pickers for picking a specific color.

File

Elements of this input type create buttons for chosing a file to upload.
Uses accept="…" to limit filetypes.

Search

Elements of this input type create text fields that specifies the input as a search field.

Checkbox

Elements of this input type turns the input into a check toggle.

Select

Elements of this input type create select boxes, aka dropdown boxes, that allows a user to pick one option of many.

Textarea

Elements of this input are an alternative to input type='text' that allows multiple lines and longer text.

Datalist

The input is linked to the datalist with the attribute "list". It has a placeholder and the datalist has 3 options.

Input attributes

  • required
    Define the input as being compulsory.
  • checked
    Whether the radio or checkbox are selected.
  • value="…"
    Puts a default value into the field.
  • placeholder="…"
    Adds a temporary, helpful hint into the field.
  • autocomplete="off"
    Disable auto completion in the field.
  • autocapitalize="none"
    Disable auto capitalization in the field. Non-standard: only works in iOS.
  • inputmode="…"
    Hint the browser to display a specific keyboard. verbatim, numeric, etc.
  • list="…"
    The id of an associated datalist
  • maxlength="…"
    For setting a maximum number of characters.
  • minlength="…"
    For setting a minimum number of characters.
  • min="…"
    Sets a minimum numerical value on range & number.
  • max="…"
    Sets a maximum numerical value on range & number.
  • step="…"
    Controls how the number will increment in range & number.
  • pattern="…"
    A JavaScript regular expression to control what is valid input.
  • multiple
    For allowing multiple entries in the field. Works for: select, email, file
  • spellcheck
    Trigger the browser to perform spell checking in the field.
  • readonly
    Stops the user from modifying the value of the field.
  • disabled
    Stops any interaction on the field.
  • autofocus
    When the page loads, this field will be focused. Be really careful, only use this when the whole purpose of the page is to fill the form.
  • cols
    Coloums width i textarea
  • rows
    Rows width i textarea

Input attributes

  • :focus
    Style an element when its keyboard focused.
  • :optional
    Style an element when it doesn’t have the required attribute.
  • :required
    Style an element when it does have the required attribute.
  • :valid
    Style an element when its contents are a acceptable (email, url, pattern, etc.)
  • :invalid
    Style an element when its contents are not acceptable.
  • :checked
    Style a checkbox or radio button when it is selected.
  • :disabled
    Style an input when it has the disabled attribute.
  • :enabled
    Style an element when it doesn’t have the disabled attribute.
  • :in-range
    Style a number or range when the selected value is within the min and max.
  • :out-of-range
    Style a number or range when the selected value is outside the min and max.
  • :indeterminate
    Style a checkbox when its set to an undetermined state by JavaScript.
